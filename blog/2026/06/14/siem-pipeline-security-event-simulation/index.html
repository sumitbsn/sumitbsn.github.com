SIEM Platform — Security Event Simulation, Detection & AI-Powered Investigation Pipeline

Jun 14th, 2026

Overview

A modular SIEM-style security analytics platform built using Python, FastAPI, Redis Streams, OCSF normalization, Sigma rules, stateful correlation, and local AI models. This project simulates how enterprise SIEM/XDR/MDR platforms ingest, normalize, correlate, detect, investigate, and enrich suspicious security activity from multiple telemetry sources.

Key Features

  • Security Event Simulator API with 100,000+ Generated Security Events
  • Vendor-Style REST Endpoints with Rate-Limited API Polling
  • Redis Streams Integration for scalable event streaming
  • OCSF Normalization Layer for standardized event format
  • Sigma Rule Detection Engine with community and custom rules
  • Stateless and Stateful Correlation Detections
  • Findings / Alert Generation and Incident Aggregation
  • MITRE ATT&CK Mapping for threat intelligence
  • Local LLM Integration (Ollama) with GPT-OSS Support
  • AI-Powered Incident Analysis and Sigma Rule Generation
  • Executive Security Reporting
  • Modular Microservice Architecture

Architecture

Event Generator (Simulates security events)
    ↓
FastAPI Simulator (Vendor-style REST APIs)
    ↓
Collector Service (Fetches events via polling)
    ↓
Raw Log Storage
    ↓
Redis Streams (Event queueing)
    ↓
OCSF Mapper (Normalization)
    ↓
Sigma Engine (Rule-based detection)
    ↓
Stateless Rules (High-risk activity detection)
    ↓
Stateful Rules (Correlation-based detection)
    ↓
Findings (Alert generation)
    ↓
Incident Builder (Aggregation & enrichment)
    ↓
MITRE Mapper (ATT&CK tactics mapping)
    ↓
AI Incident Analyzer (LLM-powered investigation)
    ↓
Executive Report (Business-ready insights)
    ↓
AI Sigma Generator (Generate new detection rules)

Project Structure

siem-platform/
├── simulator/           # Event generation & API
├── collector/           # Log collection service
├── queue/               # Redis Streams producer
├── parser/              # OCSF normalization
├── detections/          # Detection engines (Sigma, stateless, stateful)
├── findings/            # Alert writer
├── pipeline/            # Orchestration
├── ai/                  # LLM integration (Ollama, incident analysis, sigma generation)
├── alerts/              # Output files (findings, incidents, reports)
└── docker-compose.yml   # Container orchestration
